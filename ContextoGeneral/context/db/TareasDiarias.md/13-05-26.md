# Contexto BD — Publicación de barbería, URL pública y QR estable

## Objetivo

Se implementó en PostgreSQL la lógica base para que una barbería publicada tenga:

* Un `slug` público para la URL de la landing.
* Un estado `publicada`.
* Una fecha `published_at`.
* Un `qr_code` estable asociado a la barbería.
* Una función para publicar barberías.
* Una función para resolver QR.
* Una función para cargar los datos públicos de la landing.

La idea final es:

```txt
QR estable: /q/QRD1DAF700
        ↓
Resuelve barberia_id
        ↓
Redirige a /b/barberia-test-fix-20260513171519
        ↓
La landing consulta datos vivos desde BD
```

---

## Decisión técnica importante

La tabla `barberias` usa `id integer`, no `uuid`.

Se confirmó en pgAdmin con:

```sql
SELECT
  column_name,
  data_type
FROM information_schema.columns
WHERE table_schema = 'public'
AND table_name = 'barberias'
AND column_name = 'id';
```

Resultado:

```txt
id integer
```

Por eso todas las funciones y tablas relacionadas usan:

```sql
barberia_id integer
```

## 4. Tabla `qr_links`

Se creó la tabla para guardar el QR estable de cada barbería.

```sql
CREATE TABLE IF NOT EXISTS public.qr_links (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  barberia_id integer NOT NULL REFERENCES public.barberias(id) ON DELETE CASCADE,
  qr_code text NOT NULL UNIQUE,
  qr_image_url text,
  active boolean NOT NULL DEFAULT true,
  created_at timestamptz NOT NULL DEFAULT now(),
  updated_at timestamptz NOT NULL DEFAULT now()
);
```

### Significado

```txt
id            → identificador interno del registro QR.
barberia_id   → barbería asociada.
qr_code       → código estable del QR.
qr_image_url  → URL de la imagen QR, puede quedar null por ahora.
active        → permite activar/desactivar QR.
created_at    → fecha de creación.
updated_at    → fecha de actualización.
```

Ejemplo real generado:

```txt
barberia_id = 118
qr_code = QRD1DAF700
active = true
qr_image_url = null
```

El QR debe apuntar a:

```txt
/q/QRD1DAF700
```

No debe apuntar directamente a `/b/slug`, porque el slug puede cambiar en el futuro.

## 8. Función `ba_slugify`

Esta función convierte un nombre de barbería en un slug limpio.

Ejemplo:

```txt
Barbería El Paisa → barberia-el-paisa
```

SQL:

```sql
CREATE OR REPLACE FUNCTION public.ba_slugify(input_text text)
RETURNS text
LANGUAGE sql
IMMUTABLE
AS $$
  SELECT trim(both '-' FROM regexp_replace(
    lower(
      translate(
        coalesce(input_text, ''),
        'áéíóúÁÉÍÓÚñÑäëïöüÄËÏÖÜ',
        'aeiouAEIOUnNaeiouAEIOU'
      )
    ),
    '[^a-z0-9]+',
    '-',
    'g'
  ));
$$;
```

Prueba:

```sql
SELECT public.ba_slugify('Barbería El Paisa');
```

Resultado esperado:

```txt
barberia-el-paisa
```

## 9. Función `ba_generate_unique_slug`

Esta función genera un slug único para la barbería.

Si ya existe:

```txt
barberia-el-paisa
```

crea:

```txt
barberia-el-paisa-2
barberia-el-paisa-3
```

SQL:

```sql
CREATE OR REPLACE FUNCTION public.ba_generate_unique_slug(
  p_nombre text,
  p_barberia_id integer
)
RETURNS text
LANGUAGE plpgsql
AS $$
DECLARE
  base_slug text;
  final_slug text;
  counter int := 1;
BEGIN
  base_slug := public.ba_slugify(p_nombre);

  IF base_slug IS NULL OR base_slug = '' THEN
    base_slug := 'barberia';
  END IF;

  final_slug := base_slug;

  WHILE EXISTS (
    SELECT 1
    FROM public.barberias b
    WHERE b.slug = final_slug
    AND b.id <> p_barberia_id
  ) LOOP
    counter := counter + 1;
    final_slug := base_slug || '-' || counter;
  END LOOP;

  RETURN final_slug;
END;
$$;
```

## 10. Función `ba_generate_qr_code`

Genera un código QR único tipo:

```txt
QRD1DAF700
```

SQL:

```sql
CREATE OR REPLACE FUNCTION public.ba_generate_qr_code()
RETURNS text
LANGUAGE plpgsql
AS $$
DECLARE
  new_code text;
BEGIN
  LOOP
    new_code := 'QR' || upper(substr(replace(gen_random_uuid()::text, '-', ''), 1, 8));

    EXIT WHEN NOT EXISTS (
      SELECT 1
      FROM public.qr_links
      WHERE qr_code = new_code
    );
  END LOOP;

  RETURN new_code;
END;
$$;
```

Prueba:

```sql
SELECT public.ba_generate_qr_code();
```

Resultado esperado parecido a:

```txt
QR8F91AB2C
```

## 11. Función principal `ba_publicar_barberia`

Esta es la función central de publicación.

### Qué hace

```txt
1. Recibe el id de la barbería.
2. Busca la barbería.
3. Valida que exista.
4. Valida que tenga nombre.
5. Valida que tenga al menos 1 servicio activo.
6. Valida que tenga al menos 1 barbero activo.
7. Crea slug si no existe.
8. Conserva slug si ya existe.
9. Crea qr_code estable si no existe.
10. Conserva qr_code si ya existe.
11. Marca publicada = true.
12. Guarda published_at.
13. Devuelve public_path y qr_path.
```

SQL:

```sql
CREATE OR REPLACE FUNCTION public.ba_publicar_barberia(
  p_barberia_id integer
)
RETURNS jsonb
LANGUAGE plpgsql
AS $$
DECLARE
  v_barberia public.barberias%ROWTYPE;
  v_slug text;
  v_qr_code text;
  v_servicios_count int;
  v_barberos_count int;
BEGIN
  SELECT *
  INTO v_barberia
  FROM public.barberias
  WHERE id = p_barberia_id;

  IF NOT FOUND THEN
    RETURN jsonb_build_object(
      'ok', false,
      'error', 'barberia_no_encontrada'
    );
  END IF;

  SELECT count(*)
  INTO v_servicios_count
  FROM public.servicios
  WHERE barberia_id = p_barberia_id
  AND activo = true;

  SELECT count(*)
  INTO v_barberos_count
  FROM public.barberos
  WHERE barberia_id = p_barberia_id
  AND activo = true;

  IF v_barberia.nombre IS NULL OR trim(v_barberia.nombre) = '' THEN
    RETURN jsonb_build_object(
      'ok', false,
      'error', 'falta_nombre_barberia'
    );
  END IF;

  IF v_servicios_count < 1 THEN
    RETURN jsonb_build_object(
      'ok', false,
      'error', 'faltan_servicios_activos'
    );
  END IF;

  IF v_barberos_count < 1 THEN
    RETURN jsonb_build_object(
      'ok', false,
      'error', 'faltan_barberos_activos'
    );
  END IF;

  IF v_barberia.slug IS NULL OR trim(v_barberia.slug) = '' THEN
    v_slug := public.ba_generate_unique_slug(v_barberia.nombre, p_barberia_id);

    UPDATE public.barberias
    SET slug = v_slug
    WHERE id = p_barberia_id;
  ELSE
    v_slug := v_barberia.slug;
  END IF;

  SELECT qr_code
  INTO v_qr_code
  FROM public.qr_links
  WHERE barberia_id = p_barberia_id
  AND active = true
  LIMIT 1;

  IF v_qr_code IS NULL THEN
    v_qr_code := public.ba_generate_qr_code();

    INSERT INTO public.qr_links (
      barberia_id,
      qr_code,
      active
    )
    VALUES (
      p_barberia_id,
      v_qr_code,
      true
    );
  END IF;

  UPDATE public.barberias
  SET
    publicada = true,
    published_at = COALESCE(published_at, now())
  WHERE id = p_barberia_id;

  RETURN jsonb_build_object(
    'ok', true,
    'barberia_id', p_barberia_id,
    'slug', v_slug,
    'qr_code', v_qr_code,
    'public_path', '/b/' || v_slug,
    'qr_path', '/q/' || v_qr_code
  );
END;
$$;
```

## 15. Función `ba_resolver_qr`

Esta función se usa cuando alguien abre o escanea:

```txt
/q/QRD1DAF700
```

La función busca el `qr_code`, obtiene el `barberia_id`, busca el slug actual y devuelve la ruta a la landing pública.

SQL:

```sql
CREATE OR REPLACE FUNCTION public.ba_resolver_qr(
  p_qr_code text
)
RETURNS jsonb
LANGUAGE plpgsql
AS $$
DECLARE
  v_barberia_id integer;
  v_slug text;
BEGIN
  SELECT q.barberia_id, b.slug
  INTO v_barberia_id, v_slug
  FROM public.qr_links q
  JOIN public.barberias b ON b.id = q.barberia_id
  WHERE q.qr_code = p_qr_code
  AND q.active = true
  AND b.publicada = true
  LIMIT 1;

  IF v_barberia_id IS NULL THEN
    RETURN jsonb_build_object(
      'ok', false,
      'error', 'qr_no_encontrado'
    );
  END IF;

  RETURN jsonb_build_object(
    'ok', true,
    'barberia_id', v_barberia_id,
    'slug', v_slug,
    'redirect_path', '/b/' || v_slug
  );
END;
$$;
```

### Prueba

```sql
SELECT public.ba_resolver_qr('QRD1DAF700');
```

Resultado confirmado:

```json
{
  "ok": true,
  "barberia_id": 118,
  "slug": "barberia-test-fix-20260513171519",
  "redirect_path": "/b/barberia-test-fix-20260513171519"
}
```

## 16. Función `ba_get_landing_publica`

Esta función devuelve los datos vivos de la landing pública por `slug`.

Se creó una versión básica usando solo columnas existentes confirmadas.

### Qué devuelve

```txt
- Datos de la barbería.
- Servicios activos.
- Barberos activos.
```

SQL:

```sql
CREATE OR REPLACE FUNCTION public.ba_get_landing_publica(
  p_slug text
)
RETURNS jsonb
LANGUAGE sql
STABLE
AS $$
  SELECT jsonb_build_object(
    'ok', true,
    'barberia', jsonb_build_object(
      'id', b.id,
      'nombre', b.nombre,
      'slug', b.slug,
      'publicada', b.publicada
    ),
    'servicios', COALESCE((
      SELECT jsonb_agg(
        jsonb_build_object(
          'id', s.id,
          'nombre', s.nombre,
          'duracion_min', s.duracion_min,
          'precio', s.precio
        )
        ORDER BY s.id
      )
      FROM public.servicios s
      WHERE s.barberia_id = b.id
      AND s.activo = true
    ), '[]'::jsonb),
    'barberos', COALESCE((
      SELECT jsonb_agg(
        jsonb_build_object(
          'id', br.id,
          'nombre', br.nombre
        )
        ORDER BY br.id
      )
      FROM public.barberos br
      WHERE br.barberia_id = b.id
      AND br.activo = true
    ), '[]'::jsonb)
  )
  FROM public.barberias b
  WHERE b.slug = p_slug
  AND b.publicada = true
  LIMIT 1;
$$;
```

## 19. Flujo final logrado

```txt
Dueño da clic en Publicar
        ↓
Backend deberá llamar ba_publicar_barberia(barberia_id)
        ↓
BD valida nombre, servicios y barberos activos
        ↓
BD crea/conserva slug
        ↓
BD crea/conserva qr_code estable
        ↓
BD marca publicada = true
        ↓
BD devuelve public_path y qr_path
        ↓
Frontend muestra URL pública y QR
```

Flujo del QR:

```txt
Cliente escanea QR
        ↓
Abre /q/QRD1DAF700
        ↓
Backend llama ba_resolver_qr('QRD1DAF700')
        ↓
BD devuelve /b/barberia-test-fix-20260513171519
        ↓
Backend redirige a la landing pública
```

Flujo de landing:

```txt
Cliente abre /b/barberia-test-fix-20260513171519
        ↓
Frontend/backend toma el slug
        ↓
Llama ba_get_landing_publica(slug)
        ↓
BD devuelve barbería + servicios activos + barberos activos
        ↓
Landing muestra datos actualizados
```

## 21. Reglas importantes para mantener la arquitectura limpia

```txt
1. El QR no guarda servicios ni barberos.
2. La URL no guarda servicios ni barberos.
3. La BD es la fuente de verdad.
4. La landing pública siempre consulta datos vivos por slug.
5. El QR debe apuntar a /q/qr_code, no directamente a /b/slug.
6. Si el slug cambia, el QR no se rompe porque sigue asociado a barberia_id.
7. Publicar no debe congelar la landing.
8. Publicar solo crea/conserva slug, qr_code y marca publicada = true.
```
