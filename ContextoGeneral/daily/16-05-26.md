## Hipotesis iniciales que deben comprobarse hoy
1. La landing publica puede estar quedando sin `slug` util para hidratar la RPC publica bajo `/b/{slug}`.
2. El router/plugin de WordPress puede estar sirviendo HTML legacy sin inyectar correctamente el contexto publico que necesitan las plantillas.
3. Las plantillas pueden haber quedado esperando una hidratacion publica que nunca corre o falla silenciosamente.
4. El editor puede publicar bien en BD, pero la pagina publica puede seguir leyendo otra ruta o un HTML cacheado distinto al contrato nuevo.
5. El QR puede estar resolviendo bien en BD, pero apuntando a una landing que visualmente no hidrata el estado publicado.

  - se retiro el refresh previo a publicar contra `ba_get_landing_publica`.
- Razon tecnica:
  - el snapshot publico sirve para lectura publica;
  - no debe usarse para armar una nueva publicacion porque puede ir atrasado respecto al estado que el usuario acaba de editar.
- Comportamiento esperado:
  - el editor publica la coleccion actual que trae el flujo de registro/editor;
  - `ba_publicar_landing_completa` sigue siendo quien reconcilia y define altas/bajas logicas en BD;
