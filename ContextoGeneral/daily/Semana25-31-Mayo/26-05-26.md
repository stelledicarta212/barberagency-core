## Plan de Implementación: Sistema de Recuperación de Contraseña Auto-servicio

### 1) Base de Datos (PostgreSQL)
Añadir la tabla `public.password_resets` para almacenar temporalmente los tokens seguros de recuperación:
```sql
CREATE TABLE IF NOT EXISTS public.password_resets (
  id SERIAL PRIMARY KEY,
  usuario_id INT NOT NULL REFERENCES public.usuarios(id) ON DELETE CASCADE,
  token UUID NOT NULL DEFAULT gen_random_uuid(),
  expires_at TIMESTAMP WITH TIME ZONE NOT NULL,
  used_at TIMESTAMP WITH TIME ZONE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
CREATE UNIQUE INDEX IF NOT EXISTS ix_password_resets_token ON public.password_resets(token);
```

### 2) Automatización Back-end (n8n Workflows)
Crear un nuevo workflow de automatización que exponga dos endpoints REST:
- **`dashboard/recover/request` (POST):** Recibe el email ➡️ verifica que exista en la tabla de usuarios ➡️ genera y almacena un token UUID en `public.password_resets` con validez de 1 hora ➡️ envía un correo HTML premium (vía SMTP/Resend) con el enlace único: `https://barberagency-app.gymh5g.easypanel.host/restablecer-password?token=TOKEN_UUID`.
- **`dashboard/recover/reset` (POST):** Recibe `token` y `new_password` ➡️ valida la vigencia del token (comprobando que no haya expirado y `used_at IS NULL`) ➡️ hashea la nueva contraseña mediante `public.fn_password_hash(new_password)` ➡️ actualiza `password_hash` en `public.usuarios` ➡️ marca el token como utilizado asignando `used_at = NOW()` (para auditoría e invalidación).
